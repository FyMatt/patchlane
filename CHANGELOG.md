# 更新日志

## 0.0.1

- 初始版本：中文优先的 VS Code 工程 Agent。
- 支持 DeepSeek 优先接入，并提供 OpenAI、Claude、国内主流模型、本地模型和自定义接口。
- 支持 Chat / Agent 两种模式。
- Agent 可读取工作区上下文、引用文件、生成可确认 unified diff。
- 修改结果页支持实时 diff、质量审查、应用、部分应用、撤回。
- 支持页面内审批：文件读写、命令、Skill、MCP、联网搜索。
- 支持 Skill / Tool / MCP 模板生成和 stdio / HTTP MCP 基础调用，HTTP MCP 可读取 `text/event-stream` SSE 响应。
- 支持 MCP 工具、资源、Prompt 目录发现；资源读取和 Prompt 获取会走页面内审批并写入当前会话上下文。
- 支持 MCP Prompt 参数表单和 Skill / Tool / MCP 配置诊断。
- 支持 `.patchlane/patchlane.json` 项目级能力清单，团队可以把 Skill、Tool、MCP 服务随仓库共享；清单解析错误会进入设置页配置诊断。
- 支持设置页创建 Skill，自动生成 `.patchlane/skills/<id>/index.*` 并写入工作区设置。
- 支持在扩展能力工作台打开自定义脚本并测试运行 Skill、工具和 MCP。
- 支持记录本会话 Skill、工具和 MCP 的最近运行结果，包含 Agent 自动调用、设置页测试、退出码、命令和输出摘要。
- 支持 MCP 服务重连、停止、清空最近日志和运行日志展示。
- HTTP MCP/SSE 解析补齐回归测试，覆盖 plain JSON、批量 JSON-RPC、多事件 SSE、多行 `data:` 和 `[DONE]`。
- 支持联网搜索、官方文档优先搜索、GitHub 优先搜索、最新消息优先搜索。
- 支持 Agent 前文计划继承：当用户在同一会话中说“继续”“按上面的计划实现”“逐一完成所有任务”时，会提取前文计划、步骤和验收标准作为独立上下文。
- 统一模型错误归因：API Key 未配置、空响应、只返回思考内容、上下文过长、限流、网络失败和服务端错误会转换为中文可行动提示。
- 模型完全未输出 diff 时，修改结果页会保留失败状态和质量审查说明，方便判断重试、换模型、调大输出上限或拆分任务。
- 支持失败归因修正策略：模型空响应、上下文过长、patch 截断、应用冲突和验证失败会转换成下一步策略，并注入修复提示，减少盲目重试和无关大改。
- 支持自动发现验证命令、多命令验证套件、失败分类、验证失败后生成修复草稿。
- 支持安全的多轮验证修复：应用修复草稿后会再次验证，仍失败时在上限内继续生成下一轮可审查草稿，所有写入仍需人工确认。
- 支持会话持久化、任务状态提示、任务中断恢复卡片、一键重试和停止生成。
- 支持本地 VSIX 打包脚本，并补充发布清单和贡献指南。
